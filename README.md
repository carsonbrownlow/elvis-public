# elvis-public

Public static hosting layer for Carson's internal Elvis system artifacts.

## What lives here

- Dashboards / status pages that need a rendered URL
- Architecture diagrams (mermaid, SVG)
- Static HTML artifacts that need to be openable from a phone

## Why public

Internal artifacts that need a real cloud URL. Private GitHub repos can't render HTML via htmlpreview.github.io and Vercel deploy is overkill for static files. This repo is GitHub-Pages-enabled — drop a file at `foo.html` and it's reachable at `https://carsonbrownlow.github.io/elvis-public/foo.html`.

## Index

- [Elvis Status](elvis-status.html) — live snapshot of the Elvis system (regenerated by `~/bin/elvis-status.sh`)
- [Architecture Diagram](elvis-architecture.html) — full system map (mermaid-rendered)

## Pairing

This repo pairs with the private `carsonbrownlow/elvis` repo, which holds the actual system code, prompts, and DB-bound logic. Anything sensitive stays private; only artifacts safe-to-publish land here.
